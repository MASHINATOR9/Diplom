Задание 1. Представь: тебе нужно проверить, отображается ли созданный заказ в базе данных. Для этого: выведи список логинов курьеров с количеством их заказов в статусе «В доставке» (поле inDelivery = true).

Для выполнения задания необходимо подключиться к базе данных. Делать это будем следующим образом:

Открываем консоль Cygwin;
Генерируем ключи для сервера - "ssh-keygen";
Выводим ключ - "cat ~/.ssh/id_ed25519.pub";
Rлюч используем для запуска удаленного сервера;
Подключаемся к удаленному серверу - "$ ssh <имя пользователя>@<хост> -p <порт>";
Доступ к базе осуществляется с помощью команды "psql -U morty -d scooter_rent". Пароль: smith;
Создаем курьеров и заказы через запросы в Postman (используем данные из первого практического блока);
При помощи Android Studio заходим в приложение "Яндекс Самокат", авторизуемся за несколько курьеров и принимаем заказы;
Делаем SQL запрос:  SELECT с.login AS "Логин_курьера", 
                    COUNT(o.id) AS "Количество_взятых_заказов" 
                    FROM "Couriers" c JOIN "Orders" o ON c.id = o."courierId" 
                    WHERE o."inDelivery" = true 
                    GROUP BY c.login;

Скриншот запроса и результата запроса - Задание 1_1.png и Задание 1_2.png (директория /Diplom)


Задание 2. Ты тестируешь статусы заказов. Нужно убедиться, что в базе данных они записываются корректно. Для этого: выведи все трекеры заказов и их статусы. Статусы определяются по следующему правилу: Если поле finished == true, то вывести статус 2. Если поле canсelled == true, то вывести статус -1. Если поле inDelivery == true, то вывести статус 1. Для остальных случаев вывести 0.

Для выполнения Задания 2 повторяем шаги из Задания 1.

Делаем SQL запрос:  SELECT track AS "Трек_номер", 
                    (CASE WHEN finished=true THEN 2 WHEN cancelled=true THEN -1 WHEN "inDelivery"=true THEN 1 ELSE 0 END) AS "Статус" 
                    FROM "Orders";

Скриншот запроса и результата запроса - Задание 2_1.png и Задание 2_2.png  (директория /Diplom)
